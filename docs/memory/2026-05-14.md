## Decisions Made
- [Tier 1] CI baseline is `npm ci` + `npm run check` in GitHub Actions. Reasoning: the repo currently has a Vite prototype and no automated test suite, so build verification is the most useful low-noise check to catch broken changes immediately.
- [Tier 1] CI runs on pull requests and on pushes to `main` and `cursor/**` branches so feature branches receive feedback before merge.
- [Tier 1] Cloud Run runtime will use a repo-owned `server.js` static server rather than relying on a globally installed CLI package in the container. Reasoning: this keeps the production entrypoint explicit in version control and makes startup/health behavior testable locally.
- [Tier 1] Cursor cloud agents for this repo should authenticate GCP using the environment-scoped secret `GCP_SERVICE_ACCOUNT_KEY_JSON` (or an existing `GOOGLE_APPLICATION_CREDENTIALS` mount) during the environment install step. Reasoning: Cloud Run deployment from cloud agents is non-interactive, so `gcloud auth login` is not a viable long-term path.
